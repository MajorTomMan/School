# School 有机化学分子图、官能团、异构与反应内核

## 核心原则

有机化学中的名称、分子式、结构简式和键线式都应指向同一个分子图。底层不能仅比较字符串，也不能依赖一张不可计算的结构图片。

`OrganicMolecule` 保存：

- 原子身份
- 形式电荷
- 芳香性
- 显式氢
- 单键、双键、三键和芳香键
- 原子连接关系

根据价态和键级推导隐式氢，进而生成分子式、检查价态并分析碳骨架。

## 结构输入

`OrganicNotationParser` 使用受限的 SMILES 风格技术记法，支持：

- 常见元素
- 单、双、三和芳香键
- 分支括号
- 一位环闭合编号
- 芳香小写原子
- 简单方括号原子，例如 `[NH4+]`、`[O-]`
- 点号分隔的多个组分

示例：

```text
CCO          乙醇连接关系
COC          二甲醚连接关系
CC(=O)O      羧酸结构
CC(=O)OCC    酯结构
c1ccccc1     苯环
```

这是内部技术输入，不要求教材直接使用 SMILES。正式课程可以提供拖放、结构简式或键线式编辑器，再转换到同一分子图。

## 官能团

`FunctionalGroupDetector` 首批识别：

- 碳碳双键
- 碳碳三键
- 卤素
- 羟基
- 醚键
- 醛基
- 酮羰基
- 羧基
- 酯基
- 氨基
- 酰胺基
- 芳香环

识别结果保存对应原子集合，界面可以直接高亮整个官能团，而不是只显示名称。

## 碳骨架和同分异构

`CarbonSkeleton` 提取最长简单碳链以及链上的双键、三键位置，为后续基础命名提供结构依据。

`MoleculeIsomorphism` 使用原子候选约束和回溯映射比较连接关系。`OrganicIsomerAnalyzer` 区分：

- 分子式不同
- 同一个连接结构的不同写法
- 分子式相同但连接关系不同的构造异构体

本轮尚未完成顺反异构、手性中心和构象等完整立体化学判断，但能力枚举已经预留。

## 反应模板

`OrganicReactionTemplate` 明确保存：

- 反应类别
- 反应物分子图
- 生成物分子图
- 反应条件

模板创建时必须通过元素和总电荷守恒。运行时对实际反应物执行图同构匹配，不只比较输入字符串。

首个技术模板是乙烯和氢气的催化加成。它用于验证反应模板、守恒和无序反应物匹配，不代表系统会根据任意有机底物自动预测产物。

## 二维布局

`OrganicMoleculeLayout` 为小型教材分子提供确定性力导向布局数据。Compose Canvas 使用同一分子图绘制原子和不同键级。正式键线式布局以后可以在不改变分子语义的情况下替换布局器。

## 性能与边界

- 单个分子限制原子和化学键数量。
- 图同构先比较分子式、度数、键级和局部环境，再执行回溯。
- 大型碳骨架、聚合物和生物大分子不会使用无界全图搜索。
- 不做任意有机反应产物预测。
- 不做专业级系统命名、立体化学、反应机理或量子化学计算。
- 反应规则必须由具体教材章节启用并带条件。

当前内核足以为高中有机化学的结构式、官能团、构造异构和基础反应类型建立统一底层。
