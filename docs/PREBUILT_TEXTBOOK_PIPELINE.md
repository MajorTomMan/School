# 全学科教材预制流水线

## 目标

School 不把教材 PDF 或教材正文打进 APK。预制资产只保存：

- PDF SHA-256 指纹与文件别名
- 学科、阶段、卷册和出版信息
- 教材自身目录层级
- 每个可学习节点的页码范围和学习角色
- School 自有的目标、讲解框架、误区与练习描述

绑定匹配 PDF 后，应用复用同一课程树并开放教材原页；未绑定 PDF 时课程、掌握度和复习仍可使用。

## 当前批次

- 现有数学：11 本、280 个知识单元
- 新增语文、英语、日语、物理、化学：34 本、1,027 个课程节点
- 合计：45 本、1,307 个预制课程节点

45 本 PDF 共约 1.09 GB，不进入仓库。44 本具有可用文本层，40 本具有 PDF 书签目录；唯一没有文本层的高中数学必修第二册仍具有完整书签，因此不需要整本 OCR。

## 目录生成

优先级：

1. PDF 书签目录
2. 教材内置“学习内容一览”或目录表
3. 文本层目录行
4. 手机端 OCR 回退

书签祖先会保存为 `MODULE / UNIT / CHAPTER / LESSON / TOPIC` 路径，叶子节点保存页码范围。相邻书签决定上一节点的结束页，不使用固定页数切块。

## 学习角色

目录节点会标记为：

- `CORE`
- `READING`
- `WRITING`
- `LISTENING`
- `SPEAKING`
- `GRAMMAR`
- `VOCABULARY`
- `EXPERIMENT`
- `PROJECT`
- `REVIEW`
- `WHOLE_BOOK`

这些角色用于选择学科专属的学习模板，但不伪装成已经完成的精细判题器。数学继续使用确定性数学分析与题库；其他学科先提供目录、教材定位、结构化讲解和基础练习，后续再逐学科增加评价器。

## 资产

- `app/src/main/assets/prebuilt/math/index.json`：数学专属知识包
- `app/src/main/assets/prebuilt/textbooks/index-*.b64`：其他学科压缩分片目录包

生成器输出必须通过单元测试验证：教材数、课程数、槽位唯一、指纹唯一、页码边界、路径节点类型和稳定卷册 ID。
