# School 课程内容契约

## 目标

课程目录只说明“教材有哪些章节”，科学与语言内核只说明“程序会哪些能力”。两者之间需要一层课程内容契约，明确每一课：

- 内容来自教材、School 解释还是可选扩展；
- 教材依据位于哪些页；
- 使用什么可视化；
- 哪些参数可以由学习者调整；
- 使用哪些概念、操作和组件；
- 本课可以执行什么验证任务。

这层契约防止页面因为底层已经支持某项高级能力，就在教材尚未引入时提前展示。

## 内容来源

`CourseNote` 使用现有 `ContentOrigin`：

- `TEXTBOOK_QUOTE`：教材短引用，必须绑定本课页码；
- `TEXTBOOK_SUMMARY`：依据教材页面整理的摘要，必须绑定本课页码；
- `SCHOOL_EXPLANATION`：School 自有解释，不冒充教材原文；
- `OPTIONAL_EXTENSION`：可选扩展，标题必须明确包含“扩展”；
- `SIMULATION_DESCRIPTION`：说明可视化是教学模型还是理想模拟。

教材引用和摘要的页码必须落在课程 `pageStart..pageEnd` 内。当前仓库没有保存教材正文时，不允许仅凭课程标题生成伪造的教材引文。

## 可视化

`CourseVisualizationSpec` 描述可视化类型、说明、参数和能力需求。首批类型覆盖：

- 数轴、函数图像、二维和三维几何、向量；
- 运动、受力、电路、波；
- 粒子模型、化学方程式、分子；
- 细胞、生物过程和数据表；
- 文本结构、语言结构和一般过程。

每个可视化必须声明需要的 `ConceptId`、`OperationId` 和 `WidgetType`。加载课程时用 `LessonCapability.validate` 检查，未获本章许可的能力不能渲染。

## 自定义参数

`CourseParameterSpec` 支持：

- 连续数字；
- 整数；
- 开关；
- 固定选项；
- 文本输入。

数字参数必须声明合理范围和正步长；默认值必须位于范围内。选择参数必须有去重候选项，默认值必须属于候选项。课程参数描述的是教材模型允许改变的量，不等于允许用户任意修改所有条件。

## 验证任务

`CourseVerificationSpec` 为课程页面和“验证”主入口共用的验证契约。类型覆盖：

- 数学表达式和数量关系；
- 物理关系与物理模型；
- 化学式、化学方程式和有机结构；
- 生物数量关系与过程；
- 英语和日语句子；
- 图示标注与实验步骤。

验证任务同样声明所需概念和操作。底层无法确定时必须返回“不支持”或“存在歧义”，不能把启发式判断伪装成严格证明。

## 扩展边界

`LessonCapability.extensionPolicy` 控制扩展：

- `NONE`：禁止配置扩展内容；
- `NECESSARY_ONLY`：只允许帮助理解教材核心内容的必要背景；
- `OPTIONAL`：允许明确标注的进一步内容。

扩展知识不能改变教材章节的必会目标，也不能在练习中作为未讲先考的条件。

## 后续学科接入

后续 PR 按以下顺序接入：

1. 数学课程与通用表达式验证；
2. 物理课程、模型和电路验证；
3. 化学、有机化学课程与方程式验证；
4. 生物课程、图示、过程和数量关系；
5. 统一验证入口；
6. 英语、日语句子构造与可解释语法分析。

每个 PR 都从最新 `master` 建立，CI 通过后独立合并，避免把全学科改造塞入一个无法审查的巨型变更。
